CSS基础--定位

发表于 2020-11-03 更新于 2023-04-09 分类于 CSS

定位模型

  • 定位 = 定位模式 + 边偏移

  • static：静态定位

  • relative：相对定位

  • absolute：绝对定位

  • fixed：固定定位

  • sticky：磁贴定位模型

静态定位 static

静态定位 / 常规定位 / 自然定位

特点：

  1. 没有边偏移，是元素默认定位方式，按标准流特性摆放位置，相当于无定位

相对定位和绝对定位

特点：

  1. relative（相对定位）：不脱离常规流，保留所在位置，并以自己原本所在位置为基准

  2. absolute（绝对定位）：脱离常规流，以带有定位的最近的祖先元素为基准，若祖先元素均没有定位，则以浏览器为基准

  3. 在实际开发中经常会使用子绝父相，即子元素绝对定位，父元素用相对定位,这样子元素实现覆盖同时父元素可以保留位置，不影响后面的布局

  4. 绝对定位的元素无法用margin:auto实现居中

    • 需要给元素设置 top：50%，然后再 margin-top:负本身高度的一半。实现垂直居中
      先走父盒子的一半，再回自身的一半

    • 需要给元素设置 left：50%，然后再 margin-left:负本身宽度的一半。实现水平居中

固定定位 fix

是绝对定位的一种特殊形式

特点：

  1. 完全脱标 不占位置

  2. 只认浏览器可视窗口+边偏移来设置元素位置

    • 跟父元素没有任何关系
    • 不随滚动条滚动

sticky模型（CSS3）

磁贴定位 / 粘性定位 / 吸附定位

作用：relative + fixed 的完美结合，制造出吸附效果

特点：

  1. 如果产生偏移，原位置还是会在常规流中

  2. 如果它的最近父元素有滚动，那么它的偏移标尺就是最近的父元素

  3. 如果最近父元素没有滚动，那么它的偏移标尺是视口

  4. 上下左右的偏移规则

  5. top等定位属性更像是条件，而非是坐标

定位的堆叠顺序

  1. 当有多个带定位盒子在同一位置时，后来者居上，即后面的带定位盒子会覆盖前面的带定位盒子
  2. 使用 z-index 可以调整堆放顺序
    • 属性值为整数，可为正为负或为零，数值越大、盒子越靠上，如 z-index:2

定位会改变显示模式(display)

绝对定位和固定定位默认转换为行内块
由于行内块元素的宽度随内容宽度改变，很多情况下不满足开发需要，所以通常需要单写宽度
